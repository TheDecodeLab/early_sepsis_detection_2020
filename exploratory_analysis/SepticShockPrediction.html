cleanFeatureNames <- function(x) 
{
  x %>% 
    names() %>% 
    gsub(pattern = "-", replacement = "_") %>%
    gsub(pattern = " ", replacement = "_") %>%
    gsub(pattern = "/", replacement = "_") %>%
    gsub(pattern = ",", replacement = "") %>%
    gsub(pattern = "`", replacement = "") %>%
    gsub(pattern = "(", replacement = "", fixed = T) %>% # pattern to be replaced can also be enclosed in 
    gsub(pattern = ")", replacement = "", fixed = T) %>% # square brackets, no need of passing fixed argument then
    tolower()
}

names(rawFeatures_reduced_1) <- cleanFeatureNames(rawFeatures_reduced_1)

  • Filter out creatinine values recorded as "YES" instead of numeric, "UNKNOWN" gender, missing height and weight as imputing them might not be the best approach, some of the wbc's have time stamp as a value so filter them out as well.

  • Blood culture, Creatinine and wbc are character vectors with missing values not as NA but "". Following code handles that. For blood culture wherever there is any value - numeric or char, it is coded as 1 else 0. Idea was that if blood culture was taken then it should be coded as 1 irrespective of the results. Blood culture turn around is about 3 days. In case of creatinine, only the missing character values are encoded as NA. Similarly for wbc. Encode gender variable as 1 for male and 0 for female. Don't run this twice in a row as the values for blood culture will get messed up as per the code.

Synthetic sampling and class balancing¶

  • The only restriction on the data for the use of the SMOTE function implemented in bimba is that the predictors must be numeric and the target must be both the last column of the data frame that holds your data and have only two values. As long as the data abide by these restrictions, using the SMOTE function is easy: Source : Bimba github

  • SMOTE is not robust to noisy minority examples. Adding a cleaning step to the majority class before using SMOTE. This exercise removes the observations from the majority class which are miss-classified by their knn. This can be used for majority or minority class, decided based on exploration of original and smoted data.

  • Clean using ENN, double the size of the minority class using SMOTE and balance the distribution using RUS.
